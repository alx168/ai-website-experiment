# Product Spec

## Goal

Build a website through an AI-first workflow, with the repository structured so AI agents can make reliable, traceable progress.

## Primary objective

The project should demonstrate that a website can be planned, scaffolded, and improved through AI-generated changes guided by a strong repository harness.

## Non-goals

For the bootstrap phase, this project is not trying to:

- ship a complex production system immediately,
- support every possible feature from day one,
- optimize prompt logging or test automation before the core structure exists.

## Current phase

Documentation-first bootstrap.

The priority in this phase is to establish:

- a clear repository structure,
- a durable documentation system,
- clear working rules,
- and a defined path to app scaffolding.

## Success criteria for this phase

This phase is successful when:

- the repository has a clear docs structure,
- core project rules are documented,
- the intended product direction is written down,
- the technical direction is written down,
- and the project is ready for initial app scaffolding.

## Initial product boundary

The first usable version of the website should likely include:

- a homepage,
- a clear message about the experiment,
- basic navigation,
- and a small number of polished sections rather than many incomplete pages.

## Constraints

- Prefer AI-generated repository changes.
- Prefer small, reviewable diffs.
- Document meaningful architectural changes.
- Delay non-essential tooling until the core setup is stable.
